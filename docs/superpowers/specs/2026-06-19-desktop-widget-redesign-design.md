# FullCompo 桌面小组件架构改造设计文档

## 背景与目标

当前实现把每个组件作为一个独立 `Window`，导致：
- 编辑模式拖动卡顿（频繁修改 `Window.Position` 触发 OS 级重排）
- 首次打开组件堆在屏幕右上角
- 缺少全屏编辑界面
- 缺少统一的添加/删除/尺寸切换入口

本设计将架构改造为**单一大窗口 + Canvas**，参考 iOS 桌面小组件和 ClassIsland 2.1 全屏编辑模式，实现流畅的拖拽和编辑体验。

## 方案选择

**采用方案 A：单一大窗口承载所有组件。**

对比：
- 方案 A（单一大窗口 + Canvas）：拖动只改 Canvas 坐标，最流畅，全屏编辑最容易实现。
- 方案 B（独立窗口）：拖动卡顿难以根治，全屏编辑需要额外覆盖层，状态同步复杂。
- 方案 C（混合模式）：编辑/正常切换时组件状态迁移复杂，容易出 bug。

## 架构设计

### DesktopSurfaceWindow

- 全屏、无边框、透明背景、置顶、不显示在任务栏。
- 内部使用 `Canvas` 承载所有组件。
- 正常模式：组件区域可见，其余区域完全透明并点击穿透。
- 编辑模式：Canvas 上叠加半透明遮罩和网格线。

### WidgetContainer

- `Canvas` 上的每个组件容器是一个 `Border`。
- 容器内嵌组件通过 `IWidget.CreateView` 生成的视图。
- 容器负责：尺寸、位置、选中高亮、删除按钮、拖拽句柄。

### 数据模型

- `PanelConfig`：包含一个 `Widgets` 列表。
- `WidgetInstanceConfig`：
  - `Id`, `WidgetId`, `PanelId`
  - `SizeId`：尺寸类型 ID
  - `PosX`, `PosY`：桌面坐标
  - `Settings`：组件功能设置
- `WidgetSize`：新增像素尺寸 `Width` / `Height`。

## 组件尺寸系统

| ID | 名称 | 宽×高 |
|---|---|---|
| `mini-bar` | 迷你条 | 100×40 |
| `small-vbar` | 小竖条 | 40×100 |
| `small-hbar` | 小长条 | 180×40 |
| `small-circle` | 小圆 | 80×80 |
| `small-square` | 小方 | 80×80 |
| `medium-hbar` | 中横条 | 200×100 |
| `medium-square` | 中方 | 140×140 |
| `medium-vbar` | 中竖条 | 100×200 |
| `large-square` | 大方 | 220×220 |
| `large-vbar` | 大竖条 | 160×320 |
| `large-hbar` | 大横条 | 320×160 |

网格基准：80px，组件间隙 8px，拖动时自动吸附到 80px 网格。

### 各尺寸支持的功能

| 尺寸 | 支持功能 |
|---|---|
| 小圆 / 小方 | 日期、天气、启动器 |
| 迷你条 / 小竖条 / 小长条 | 日期、天气、自定义文本 |
| 中横条 | 日期、时钟、天气、搜索、启动器、自定义文本 |
| 中方 / 中竖条 | 日期、天气、便签、自定义文本 |
| 大方 / 大竖条 / 大横条 | 日期、时钟、天气、便签、搜索、自定义文本 |

## 默认布局

首次运行完成后，在桌面左上角创建：

```
(0,0)      日期(中横条 200×100)  时钟(中方 140×140)
(0,108)    天气(中横条 200×100)  搜索(中横条 200×100)
```

组件间距 8px。

## 编辑模式

### 进入方式

- 右键托盘图标 → "编辑模式"
- 组件右键菜单 → "编辑桌面"

### 编辑界面

- 全屏半透明深色遮罩 `#60000000`
- 淡色网格线：每 80px 一条主线，每 20px 一条细线
- 组件显示白色半透明边框
- 选中组件：边框变蓝色 `#0078D4`，右上角显示 `×` 删除按钮

### 底部工具栏

```
↺ 重置视图   🎨 外观   ➕ 添加组件   🗑 清空桌面   ↩ 撤销   ↪ 重做   ✓ 完成
```

### 交互

- 点下并移动 = 拖拽组件，自动吸附网格
- 单击（不移动）= 选中组件，右侧面板显示"选择功能"
- 右键组件 = 上下文菜单（切换尺寸、删除、编辑）

### 添加组件流程

1. 点击底部 "添加组件"
2. 右侧滑出"尺寸库"面板，分三栏：小类型 / 中类型 / 大类型
3. 选择一个尺寸 → 桌面中心添加一个**空占位组件**
4. 自动选中该占位组件 → 右侧面板显示"选择功能"
5. 选择功能后，组件渲染对应内容；如有设置项，显示设置表单

### 删除组件

- 单个删除：选中后点击右上角 `×`，或右键菜单删除
- 清空桌面：底部工具栏 "清空桌面"，需二次确认

## 组件视觉样式

- 背景：主题色 + 92% 不透明度
- 圆角：18px（小圆组件用 50%）
- 阴影：`0 4px 12px rgba(0,0,0,0.15)`
- 边框：1px 半透明白色/黑色
- 文字：根据主题自动适配颜色
- 空占位组件：显示 "+" 图标和尺寸名称

## 首次欢迎页

三步向导：

1. **欢迎**：标题 + 简介 + 下一步
2. **外观主题**：浅色 / 深色 / 毛玻璃，实时预览
3. **完成**：点击"开始使用"后创建默认组件布局

## 保存与状态

- 退出编辑模式时自动保存所有组件位置、尺寸、功能、设置。
- 撤销/重做栈仅在编辑模式期间有效，退出后清空。
- 配置文件格式保持 JSON，向后兼容旧字段（旧字段读取后迁移）。

## 性能要点

- 拖动时不保存，只在 `PointerReleased` 时保存。
- 使用 `Canvas.SetLeft` / `Canvas.SetTop` 而不是 `Window.Position`。
- 编辑模式遮罩和网格线使用轻量级 `Rectangle` 或 `Border`。
- 组件视图在尺寸/功能变化时重建，主题变化时只更新颜色。
